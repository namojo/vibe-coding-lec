# 사례 9-1: React 업무 관리 대시보드 - 프롬프트 가이드

이 가이드는 AI(Claude, ChatGPT 등)를 활용하여 React 기반 업무 관리 대시보드를 단계적으로 만드는 프롬프트 설계 방법을 안내합니다.

---

## 프로젝트 개요

- **목표**: 팀 업무를 등록/관리하는 칸반 보드 형태의 대시보드
- **기술 스택**: React 18 + Vite, CSS Modules 또는 Tailwind CSS
- **주요 기능**: 칸반 보드(할일/진행중/완료), 업무 추가/수정/삭제, 드래그앤드롭, 필터/검색, 통계

---

## 단계별 프롬프트

### 1단계: 프로젝트 초기 설정

```
React + Vite 프로젝트를 생성해주세요.
- 프로젝트 이름: task-dashboard
- Tailwind CSS 설정 포함
- 기본 폴더 구조: components/, hooks/, utils/, data/
- 한국어 기본 설정 (lang="ko")
- 더미 데이터 파일 (data/tasks.js) 포함
```

### 2단계: 데이터 구조 설계

```
업무 관리에 필요한 데이터 구조를 설계해주세요.

업무(Task) 필드:
- id, 제목, 설명, 상태(할일/진행중/완료), 담당자
- 우선순위(높음/보통/낮음), 마감일, 생성일, 태그[]

상태 관리는 useReducer + Context API로 구현해주세요.
10개의 더미 업무 데이터도 만들어주세요.
```

### 3단계: 레이아웃 컴포넌트

```
대시보드 레이아웃을 만들어주세요.

구조:
- Header: 로고, 검색바, 사용자 아바타
- Sidebar: 네비게이션 (대시보드, 칸반보드, 캘린더, 설정)
- Main: 라우터 영역

다크/라이트 모드 토글 버튼도 포함해주세요.
반응형으로 모바일에서는 사이드바가 햄버거 메뉴로 변환되어야 합니다.
```

### 4단계: 칸반 보드

```
칸반 보드 페이지를 만들어주세요.

3개 컬럼: 할일 | 진행중 | 완료
각 컬럼에 업무 카드를 표시하고, 카드에는 다음을 표시:
- 제목, 담당자 아바타, 우선순위 배지, 마감일
- 태그 칩

카드 클릭 시 상세 모달 표시
각 컬럼 상단에 업무 수 표시
"+ 새 업무" 버튼으로 업무 추가

드래그앤드롭은 react-beautiful-dnd 또는
HTML5 drag API로 구현해주세요.
```

### 5단계: 업무 추가/수정 폼

```
업무 추가 및 수정 모달 폼을 만들어주세요.

필드: 제목(필수), 설명(textarea), 담당자(드롭다운),
      우선순위(라디오), 마감일(date picker), 태그(멀티 입력)

유효성 검증: 제목 필수, 마감일은 오늘 이후
저장 시 상태 업데이트 (Context dispatch)
수정 모드에서는 기존 데이터 로딩
삭제 버튼 (확인 다이얼로그 포함)
```

### 6단계: 필터와 검색

```
업무 필터링 및 검색 기능을 추가해주세요.

- 상단 검색바: 제목/설명 전문 검색
- 필터 옵션: 담당자, 우선순위, 마감일 범위
- 활성 필터를 태그 형태로 표시 (클릭으로 제거)
- 정렬: 마감일순, 우선순위순, 생성일순
- 필터 결과가 없을 때 빈 상태 메시지 표시

useMemo로 필터링 성능을 최적화해주세요.
```

### 7단계: 통계 대시보드

```
대시보드 메인 페이지에 통계를 표시해주세요.

요약 카드:
- 전체 업무 수, 완료율, 이번 주 마감, 지연 업무

차트 (recharts 또는 Chart.js):
- 상태별 업무 비율 (도넛 차트)
- 주간 완료 추이 (선 차트)
- 담당자별 업무량 (수평 막대)

최근 활동 타임라인도 추가해주세요.
```

### 8단계: 로컬 저장 및 마무리

```
다음 기능을 추가하여 프로젝트를 완성해주세요.

- localStorage로 데이터 영속화 (커스텀 훅 useLocalStorage)
- 데이터 내보내기 (JSON 다운로드)
- 키보드 단축키: N(새 업무), /(검색), Esc(모달 닫기)
- 토스트 알림 (업무 추가/수정/삭제 시)
- 빈 상태 일러스트 및 온보딩 가이드
- README.md 작성
```

## 프롬프트 작성 팁

### 좋은 프롬프트의 조건
1. **구체적인 기술 스택 명시** - "React" 보다 "React 18 + Vite + Tailwind CSS"
2. **기능 단위로 분리** - 한번에 전체를 요청하지 말고 단계별로
3. **UI 세부사항 명시** - 레이아웃, 색상, 반응형 기준점
4. **데이터 구조 선정의** - 컴포넌트 구현 전에 데이터 모델을 먼저 요청
5. **제약 조건 포함** - "외부 라이브러리 최소화", "TypeScript 사용"

### 반복 개선 프롬프트 예시

```
방금 만든 칸반 보드에서 다음을 수정해주세요:
1. 카드 호버 시 그림자 효과 추가
2. 빈 컬럼에 "업무를 드래그하여 추가하세요" 안내 메시지
3. 우선순위 "높음"인 카드에 왼쪽 빨간색 보더 추가
```

### 디버깅 요청 예시

```
다음 에러가 발생합니다. 원인과 해결 방법을 알려주세요:

[에러 메시지 붙여넣기]

관련 코드:
[코드 붙여넣기]

예상 동작: 카드를 드래그하면 컬럼 간 이동
실제 동작: 드래그 시작 후 드롭 시 원위치로 돌아감
```
