사례 7-1: Google Sheets 연동 자동화

Gemini 중급 25분

Google Sheets 데이터를 읽어와 시각화하거나 보고서로 변환하는 도구를 만듭니다.

Step 1: Sheets 데이터 뷰어 만들기

프롬프트 
Google Sheets 데이터를 가져와서 표시하는 웹 도구를 만들어주세요.

기능:
- Google Sheets URL 입력 필드
- "데이터 가져오기" 버튼
- 가져온 데이터를 테이블로 표시
- 데이터를 차트로 시각화

구현 방법:
- Google Sheets의 "웹에 게시" 기능을 활용
- 또는 데모용으로 하드코딩된 예제 데이터 제공

예제 데이터 (기본 표시):
월,민원건수,처리율(%),평균처리일
1월,234,95.2,3.2
2월,289,93.8,3.5
3월,312,96.1,2.8
4월,278,97.3,2.5
5월,345,94.5,3.1
6월,298,98.0,2.3

표시:
- 상단: 요약 카드 (총 건수, 평균 처리율, 평균 처리일)
- 중단: 데이터 테이블 (정렬 가능)
- 하단: 추이 차트 (민원건수 막대 + 처리율 선)

Chart.js 사용. 깔끔한 대시보드 디자인.
예상 결과

요약 카드, 정렬 가능한 테이블, 복합 차트가 포함된 대시보드가 생성됩니다.

Step 2: 자동 새로고침과 알림

프롬프트 
다음 기능을 추가해주세요:

1. 데이터 자동 새로고침 (30초/1분/5분 간격 선택)
2. 마지막 업데이트 시간 표시
3. 처리율이 95% 미만이면 빨간색 경고 표시
4. 데이터 변화량 표시 (전월 대비 증감, 화살표 아이콘)
5. 테이블 열 클릭 시 정렬 (오름차순/내림차순 토글)

Step 3: 보고서 내보내기

프롬프트 
보고서 내보내기 기능을 추가해주세요:
1. "보고서 생성" 버튼 — 데이터, 차트, 요약을 포함한 보고서 페이지
2. 인쇄 최적화 (A4, 깔끔한 레이아웃)
3. 보고서에 자동 날짜/시간 삽입
4. CSV 다운로드 버튼

사례 7-2: 공공데이터 API로 대시보드 만들기

Antigravity 중급 25분

공공데이터포털의 실시간 API를 활용하여 인구 통계 대시보드를 만듭니다.

개발자 트랙

Step 1: 공공데이터 API 연동

Antigravity에 접속합니다.

프롬프트 
공공데이터포털 API를 활용한 대시보드 웹페이지를 만들어주세요.

주제: 대한민국 인구 통계 대시보드

참고: 실제 API 호출은 CORS 문제가 있을 수 있으므로,
데모용으로 실제 공공데이터를 기반으로 한 하드코딩 데이터를 사용해주세요.

데이터 (2023년 기준):
지역,인구수,전년대비(%),고령화율(%),출생률
서울,"9,428,372",-0.8,17.2,0.55
부산,"3,338,774",-1.2,20.1,0.72
대구,"2,385,412",-0.9,18.5,0.78
인천,"2,980,552",0.3,14.8,0.82
대전,"1,452,251",-0.5,15.9,0.76
세종,"388,045",5.2,9.1,1.12

대시보드 구성:
- 전국 요약 카드 (총인구, 평균 고령화율, 평균 출생률)
- 지역별 인구 막대 차트
- 고령화율 수평 막대 차트 (경고색 표시: 20% 이상 빨강)
- 지역 선택 드롭다운 → 상세 정보 카드

Chart.js 사용. 반응형 디자인.

Step 2: 인터랙티브 필터링

프롬프트 
인터랙티브 필터 기능을 추가해주세요:

1. 지역 멀티 선택 체크박스 (전체 선택/해제)
2. 정렬 기준 선택 (인구수, 고령화율, 출생률)
3. 차트 유형 전환 (막대, 선, 레이더)
4. 데이터 테이블에서 행 클릭 시 해당 지역 하이라이트
5. "비교 모드" — 2개 지역을 선택하여 나란히 비교

팁 & 트러블슈팅

  • Google Sheets의 "파일 → 웹에 게시" 기능을 활용하면 외부에서 데이터를 읽을 수 있습니다
  • 대시보드 HTML 파일을 팀원들과 공유하면 같은 데이터를 시각적으로 볼 수 있습니다
  • 사례 7-1(Google Sheets)만으로도 충분히 유용합니다. 사례 7-2는 개발 경험이 있는 분께 추천합니다

공공데이터포털(data.go.kr)에서 다양한 API를 탐색해보세요.

심화 프롬프트: 프록시 서버 
Express.js 서버를 만들어서 공공데이터 API를 프록시해줘.
/api/population 엔드포인트로 요청하면
서버에서 공공데이터 API를 호출하고 결과를 반환하도록 해줘.

트러블슈팅

Sheets가 "웹에 게시" 상태인지 확인하세요. 공유 설정이 "링크가 있는 모든 사용자"로 되어 있는지 확인하세요. CORS 문제가 있을 수 있습니다.

API 키가 올바른지 확인하세요. data.go.kr에서 해당 API를 '활용 신청'했는지 확인하세요 (승인까지 시간 소요). 브라우저에서 직접 호출 시 CORS 오류가 발생할 수 있습니다.

Chart.js는 데이터 변경 후 chart.update()를 호출해야 합니다. "데이터 변경 시 차트가 자동 업데이트되도록 수정해줘"라고 요청하세요.
