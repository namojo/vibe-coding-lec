개요

학습 목표

CLI 기반 코딩(Claude Code), 파일 자동화, 알림봇 등 개발자가 AI를 실무 개발에 통합하는 방법을 익힌다

이런 분께 추천합니다:

  • 터미널/명령줄에 익숙한 개발 경험자
  • AI를 기존 개발 워크플로에 통합하고 싶은 분

사전 준비

Claude Code 설치 완료, 터미널 기본 사용법 숙지, Node.js v18 이상

사례 8-1: 파일 정리 자동화 도구

Claude 중급 20분

지정 폴더의 파일을 확장자, 날짜, 크기 등 기준으로 자동 분류하는 CLI 도구를 만듭니다.

Step 1: 프로젝트 시작

터미널을 열고 프로젝트를 생성합니다.

mkdir file-organizer && cd file-organizer
claude
Claude Code 프롬프트 
파일 정리 자동화 CLI 도구를 Node.js로 만들어줘.

기능:
- 지정된 폴더의 파일을 확장자별로 하위 폴더에 분류
  - 이미지 (jpg, png, gif, svg) → images/
  - 문서 (pdf, doc, docx, hwp, xlsx) → documents/
  - 코드 (js, py, html, css) → code/
  - 기타 → others/
- 실행 전에 미리보기 모드: 어떤 파일이 어디로 이동할지 표시
- 실행 확인 후 실제 이동
- 이동 로그를 log.txt에 저장

사용법:
node organize.js /path/to/folder        # 미리보기
node organize.js /path/to/folder --run  # 실제 실행

색상 있는 터미널 출력 (chalk 라이브러리 없이 ANSI 코드 사용)
예상 결과

organize.js 파일과 package.json이 생성됩니다. 폴더 경로를 인자로 주면 파일 분류 미리보기를 보여줍니다.

Step 2: 고급 분류 옵션 추가

Claude Code 프롬프트 
다음 기능을 추가해줘:

1. --by-date 옵션: 파일을 생성 날짜 기준 폴더(YYYY-MM)로 분류
2. --by-size 옵션: 파일 크기 기준 분류 (small <1MB, medium <100MB, large 100MB+)
3. --dry-run 옵션: 미리보기만 (기본값)
4. --recursive 옵션: 하위 폴더까지 탐색
5. 진행률 표시 (현재/전체 파일 수)
6. 중복 파일 감지 (같은 이름 존재 시 뒤에 _1, _2 추가)

Step 3: 설정 파일 지원

Claude Code 프롬프트 
JSON 설정 파일을 지원하도록 해줘:

1. organize.config.json 파일로 분류 규칙 커스터마이징
2. --init 옵션으로 기본 설정 파일 생성
3. README.md에 사용법 문서 자동 생성

사례 8-2: Slack/Teams 알림봇 만들기

Claude 중급 20분

특정 이벤트(일정, 마감일 등) 발생 시 Slack/Teams로 알림을 보내는 봇을 만듭니다.

Step 1: 알림봇 기본 구조

프롬프트 
Slack Webhook을 사용하는 알림봇을 Node.js로 만들어줘.

기능:
1. 알림 메시지 전송 함수
   - 제목, 내용, 긴급도(info/warning/error)를 인자로 받음
   - Slack Incoming Webhook URL로 POST 요청
   - 긴급도에 따라 색상 코딩

2. 일정 알림 기능
   - schedule.json 파일에서 일정 목록 로드
   - 매일 아침 9시에 오늘 일정 알림 전송
   - D-1, D-day 알림

3. 마감일 모니터링
   - deadlines.json에서 마감일 목록 로드
   - D-3, D-1, D-day에 자동 알림

실행: node bot.js
Webhook URL은 환경변수(SLACK_WEBHOOK_URL)에서 읽도록 해줘.

Step 2: 웹 인터페이스 추가

프롬프트 
알림봇에 웹 관리 페이지를 추가해줘:

1. Express.js 기반 간단한 웹 서버 (포트 3000)
2. 관리 페이지에서:
   - 현재 등록된 일정/마감일 목록 보기
   - 새 일정/마감일 추가 폼
   - 삭제 버튼
   - "지금 알림 보내기" 테스트 버튼
3. 알림 발송 이력 (최근 20개)

Step 3: Teams 지원 추가

프롬프트 
Microsoft Teams Webhook도 지원하도록 확장해줘:

1. 환경변수로 플랫폼 선택: NOTIFICATION_PLATFORM=slack 또는 teams
2. Teams Adaptive Card 형식으로 메시지 포맷팅
3. Slack과 Teams 동시 발송 옵션 (NOTIFICATION_PLATFORM=both)
4. .env.example 파일에 설정 예시 작성

사례 8-3: 반복 보고 자동화 파이프라인

Antigravity 고급 20분

데이터 수집 → 분석 → 보고서 생성 → 알림까지 자동화하는 파이프라인을 만듭니다.

Step 1: 파이프라인 기본 구조

Antigravity에 접속합니다.

프롬프트 
주간 보고 자동화 파이프라인을 Node.js로 만들어주세요.

파이프라인 단계:
1. 데이터 수집: data/ 폴더에서 CSV 파일들 읽기
2. 데이터 처리: 통계 계산 (합계, 평균, 최대, 최소, 전주 대비 변화율)
3. 보고서 생성: HTML 보고서 파일 생성 (표 + Chart.js 차트)
4. 알림: 보고서 생성 완료 메시지 콘솔 출력

실행: node pipeline.js
결과: reports/weekly_report_2026_w12.html 생성

각 단계의 시작/완료를 로그로 출력해주세요.

Step 2: 스케줄링과 히스토리

프롬프트 
파이프라인에 다음 기능을 추가해주세요:

1. --schedule 옵션: node-cron으로 매주 월요일 오전 9시 자동 실행
2. 보고서 히스토리 관리: history.json에 생성 이력 저장
3. 이전 보고서와 비교 기능: 전주 대비 변화 하이라이트
4. 에러 재시도: 실패 시 3회까지 자동 재시도 (30초 간격)
5. 상태 대시보드: http://localhost:3001에서 파이프라인 상태 확인

팁 & 트러블슈팅

Claude Code 고급 활용법

  • CLAUDE.md 파일: 프로젝트 루트에 CLAUDE.md를 만들면 Claude Code가 프로젝트 컨텍스트를 자동으로 파악합니다
  • 멀티 파일 작업: "이 프로젝트의 전체 구조를 분석하고, 리팩토링 계획을 세워줘"
  • 테스트 생성: "이 함수들에 대한 Jest 테스트를 작성해줘"
  • Git 통합: "변경 사항을 커밋하고 PR 설명을 작성해줘"
심화 프롬프트: Docker 
이 알림봇을 Docker 컨테이너로 패키징해줘.
Dockerfile, docker-compose.yml을 만들고
환경변수는 .env 파일에서 읽도록 해줘.

트러블슈팅

현재 디렉토리에 쓰기 권한이 있는지 확인하세요. Claude Code를 프로젝트 폴더 안에서 실행했는지 확인하세요.

Webhook URL이 올바른지 확인하세요 (https://hooks.slack.com/services/...). 환경변수가 설정되었는지 확인: echo $SLACK_WEBHOOK_URL

프로세스가 백그라운드에서 계속 실행 중이어야 합니다. pm2나 forever를 사용하여 프로세스를 데몬으로 실행하세요: npm install -g pm2 && pm2 start pipeline.js
