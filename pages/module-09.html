개요

학습 목표

앞서 배운 기술을 종합하여 실제 서비스 수준의 프로젝트를 만든다. React 대시보드, 챗봇, 실시간 모니터링 등을 직접 구축한다.

이런 분께 추천합니다:

  • 웹 개발 경험이 있는 분
  • AI를 활용한 프로젝트 개발 전체 흐름을 체험하고 싶은 분
  • 실무에 바로 배포할 수 있는 수준의 결과물을 원하는 분
사전 준비

Claude Code 설치 완료, Node.js v18 이상, Git 기본 사용법 숙지, (선택) React 기본 개념

사례 9-1: React 기반 관리 대시보드

Claude 고급 25분

React로 데이터 관리 대시보드를 처음부터 끝까지 만듭니다. 사이드바, 차트, 데이터 테이블, CRUD 기능 포함.

Step 1: 프로젝트 생성

mkdir admin-dashboard && cd admin-dashboard
claude
Claude Code 프롬프트 
React + Vite로 관리 대시보드를 만들어줘.

페이지 구성:
1. 대시보드 홈 (/)
   - 요약 카드 4개 (총 사용자, 신규 가입, 활성 프로젝트, 완료율)
   - 월별 추이 선 차트 (recharts 라이브러리)
   - 최근 활동 목록

2. 사용자 관리 (/users)
   - 사용자 테이블 (이름, 이메일, 부서, 가입일, 상태)
   - 검색 필터 + 새 사용자 추가 모달 + 수정/삭제

3. 설정 (/settings)
   - 프로필 수정 폼 + 알림 설정 토글

공통:
- 왼쪽 사이드바 (로고, 메뉴, 접기 버튼)
- 상단 헤더 (검색바, 알림, 프로필)
- 다크/라이트 모드 토글

데이터는 하드코딩된 mock 데이터 사용.
깔끔한 모던 스타일.
예상 결과

npm run dev로 실행하면 사이드바 네비게이션, 차트, 데이터 테이블이 포함된 완전한 관리 대시보드가 브라우저에 나타납니다.

Step 2: 테이블 기능 강화

Claude Code 프롬프트 
사용자 관리 테이블을 강화해줘:

1. 열 헤더 클릭 시 정렬 (오름/내림 토글)
2. 페이지네이션 (한 페이지 10개)
3. 체크박스로 다중 선택 → 일괄 삭제
4. 상태 필터 드롭다운
5. 행 클릭 시 상세 정보 사이드 패널
6. CSV 내보내기 버튼

Step 3: 빌드 및 배포 준비

Claude Code 프롬프트 
배포 준비를 해줘:

1. npm run build로 프로덕션 빌드
2. GitHub Pages 배포를 위한 설정 (vite.config.js의 base 경로)
3. 배포 스크립트: npm run deploy
4. README.md에 프로젝트 설명, 실행 방법 작성

사례 9-2: 챗봇 프로토타입 만들기

Claude 고급 20분

AI 챗봇 인터페이스를 만들고, 규칙 기반 응답 시스템을 구현합니다.

Step 1: 챗봇 UI 만들기

프롬프트 
NIA 업무 도우미 챗봇의 프로토타입을 만들어주세요.

UI 구성:
- 상단: 챗봇 이름 ("NIA 업무 도우미"), 온라인 상태 표시
- 중앙: 대화 영역 (말풍선 형태)
- 하단: 메시지 입력창 + 전송 버튼
- 빠른 답변 버튼 (자주 묻는 질문)

응답 규칙 (JavaScript 키워드 매칭):
- "인사" → "안녕하세요! NIA 업무 도우미입니다."
- "회의실" → "회의실 예약은 사내 포털 > 시설예약에서 가능합니다."
- "휴가" → "휴가 신청은 HR 시스템에서 진행해주세요."
- "식당", "메뉴" → "오늘의 구내식당 메뉴는..."
- 매칭 안 됨 → "죄송합니다. 해당 질문에 대한 답변을 찾지 못했습니다."

빠른 답변 버튼: "회의실 예약", "휴가 신청", "오늘 식단", "연락처 조회"

디자인: 모바일 메신저 앱 느낌, 최대 너비 450px
입력 시 "입력 중..." 표시 후 1초 딜레이로 응답

Step 2: 대화 기능 강화

프롬프트 
챗봇 기능을 강화해주세요:

1. 대화 히스토리 저장 (localStorage)
2. "대화 내보내기" 버튼 — 텍스트 파일 다운로드
3. 봇 응답에 관련 링크 포함
4. 이모지 반응 버튼 (각 봇 메시지에)
5. "도움이 되었나요?" 피드백 수집
6. 피드백 통계 패널

Step 3: AI API 연동 준비

프롬프트 
실제 AI API 연동을 위한 구조를 추가해주세요:

1. 설정 패널에 "AI 모드" 토글 (규칙 기반 ↔ AI 기반)
2. AI 모드 시 API 키 입력 필드
3. AI 모드 코드 (주석 처리):
   - Claude API 호출 함수
   - 시스템 프롬프트: "당신은 NIA 업무 도우미입니다..."
   - 대화 히스토리를 컨텍스트로 전달
4. API 키 없으면 자동으로 규칙 기반 모드로 전환

사례 9-3: 실시간 통계 모니터링 페이지

Claude 고급 15분

실시간으로 갱신되는 통계 모니터링 대시보드를 만듭니다. 다크 테마, 실시간 차트 포함.

Step 1: 실시간 대시보드 만들기

프롬프트 
실시간 통계 모니터링 대시보드를 만들어주세요.

주제: "NIA 디지털 서비스 모니터링"

표시할 지표:
1. 실시간 접속자 수 (카운터, 매초 랜덤 변동)
2. 서버 상태 (정상/주의/장애, 랜덤 시뮬레이션)
3. API 응답 시간 (ms, 실시간 선 차트로 최근 30초 표시)
4. 금일 총 요청 수 (카운트업)
5. 에러율 (%, 게이지 차트)
6. 서비스별 상태 목록

레이아웃:
- 상단: 요약 카드 4개
- 중단: 실시간 응답시간 차트 + 서비스 상태 목록
- 하단: 최근 이벤트 로그

시뮬레이션: setInterval로 매초 데이터 갱신
디자인: 다크 테마 (어두운 배경, 네온 포인트 컬러)

Step 2: 알림 및 이력 기능

프롬프트 
다음 기능을 추가해주세요:

1. 장애 발생 시 화면 상단에 빨간색 경고 배너 (깜빡임 효과)
2. 장애 이력 테이블
3. 알림 설정 패널 (각 지표의 임계값 설정)
4. 시뮬레이션 속도 조절 (1x, 2x, 5x)
5. 시뮬레이션 일시정지/재개 버튼
6. 전체 화면 모드

팁 & 트러블슈팅

프로젝트 확장 아이디어

  • 관리 대시보드: 백엔드 API(Express.js + SQLite)를 추가하여 실제 CRUD 구현
  • 챗봇: Claude API를 연결하여 실제 AI 기반 응답, RAG 적용
  • 모니터링: WebSocket을 사용한 실시간 데이터 스트리밍
심화 프롬프트: CI/CD 
GitHub Actions 워크플로우를 만들어줘:
- main 브랜치 push 시 자동 빌드
- 테스트 실행
- GitHub Pages에 자동 배포

트러블슈팅

node_modules가 있는지 확인: ls node_modules. 없으면: npm install. 포트 충돌 시: npx vite --port 3001

브라우저 개발자 도구(F12) → Console 탭에서 에러 메시지를 확인하세요. Claude Code에게 에러 메시지를 복사하여 해결을 요청하세요.

useEffect 내에서 setInterval 정리(cleanup)가 제대로 되는지 확인하세요.

vite.config.js의 base 설정이 저장소 이름과 일치하는지 확인하세요. React Router를 사용한다면 HashRouter로 변경하세요.
